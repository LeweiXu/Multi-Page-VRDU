\section{Introduction}

\begin{itemize}
    \item Documents are the primary medium through which organisations record and communicate information, with many critical real-world document types such as contracts, financial reports, scientific papers, technical manuals, textbooks, inherently spanning multiple pages and containing interleaved text, tables, figures and complex layouts \cite{tito2023hierarchical, borchmann2025arctic}
    \item Manually extracting information is time-consuming, expensive, and prone to human error; development of automated DocVQA systems that can answer natural-language queries over document images aims to solve this
    \item Significant progress has been made on single-page docVQA, with current SOTA models on-par if not exceeding human level performance, but vast majority of real-world documents extend beyond single page \cite{tito2023hierarchical}
    \item Evidence required to answer a question is often located on multiple pages scattered throughout the document, requiring a model to navigate, retrieve and reason across the entire document \cite{blau2024gram, borchmann2025arctic}
    \item challenges from single-page to multi-page VQA: 1. input token count scales linearly (or worse) quickly exceeding context window of existing LLMs and MLLMs, visually rich pages containing high resolution figures/tables/charts contribute to this. 2. capturing cross-page semantic and logical dependencies requires explicit long-range reasoning. 3. Explainability becomes more difficult as models need to also pinpoint how they arrived at the answer via answer page/passage retrieval.
    \item Early mult-page approaches adapt pre-trained language models to document domain treating OCR'd documents as a long text sequence however these methods ignored visual content.
    \item Subsequent work introduced hierarchical page-level architectures \cite{tito2023hierarchical} and global-reasoning token mechanisms \cite{blau2024gram} to enable cross-page reasoning and interaction achieving strong results but struggled with longer documents
    \item Unlike existing surveys that focus on single-page and broad document VRDU \cite{ding2025survey}, this survey focuses exclusively on the multi-page frontier for document question answering.
    \item we survey 30 (could be more) representative models spanning both OCR-dependent and OCR-free approaches.
\end{itemize}

\section{Framework Architecture Overview and Trends}
Previous VRDU surveys often organise frameworks by OCR dependency, separating OCR-based models that consume text and layout boxes from OCR-free models that process document images directly \cite{huang2022layoutlmv3, ding2025survey}. In MP-VRDU, this distinction is less binary. Many systems use OCR-derived text as one signal for retrieval, grounding, navigation, or verification rather than as the sole document representation \cite{sun2025docagent, wang2025vidorag}. Conversely, OCR-free inference does not imply full independence from OCR, as visual-language backbones are often trained or aligned with OCR-derived text recognition data \cite{tanaka2025vdocrag, hu2024mplug}. We therefore treat OCR dependency as a secondary descriptor in Table~\ref{tab:model_survey_arch} and organise MP-VRDU frameworks by architectural trend, covering page-to-document transformers, backbone-centric MLLM adaptations, retriever-generator systems, and agentic pipelines.

% Table~\ref{tab:model_survey_arch} compares MP-VRDU frameworks along two axes: OCR dependency and document-level architecture. We include OCR dependency not as a general VRDU taxonomy, which is covered in prior surveys \cite{ding2025survey}, but to highlight its distinct role in the multi-page setting. The second axis captures how each framework controls the challenges presented in visually rich long document understanding.

It can support sparse retrieval, page grounding, bounding-box references, and textual verification, while visual encoders and VLMs handle image-based evidence \cite{han2025mdocagent, zhu2025simple, wang2025vidorag}. \subsection{Page-to-Document Transformer}
This family extends page-level document encoders to the multi-page setting by encoding pages as separate structured units and adding document-level information exchange. Rather than flattening all tokens and visual patches into one sequence, these models preserve page boundaries while connecting pages through summary tokens \cite{tito2023hierarchical}, global document tokens with local-global attention \cite{blau2024gram}, chunked sparse attention over long OCR streams \cite{borchmann2025arctic}, or recurrent memory passed across pages \cite{dong2024multi}. This structure provides an inductive bias for MP-VRDU, enabling evidence aggregation across pages without discarding page identity. However, cross-page reasoning remains bounded by the exchange mechanism, as summaries and memories can lose fine-grained evidence, sparse attention may miss distant dependencies, and computation generally grows with document length.

% This family uses a learned document encoder to separately encode individual pages, then introduces a document-level mechanism for information exchange across page representations. Hi-VT5 encodes each page independently and passes question-conditioned page tokens to the decoder, but its cross-page reasoning is delayed until after page-level compression \cite{tito2023hierarchical}. GRAM addresses this bottleneck by injecting document-global tokens into the encoder so pages can exchange information during representation learning, although its cost still increases with document length and optional compression can reduce answer quality \cite{blau2024gram}. RM-T5 instead carries recurrent memory tokens from page to page, making document aggregation more parameter-efficient, but fixed sequential memory can over-compress or forget earlier evidence \cite{dong2024multi}. Arctic-TILT pushes the same direction to much longer OCR-based inputs through layer-wise multimodal fusion and chunked sparse attention, but remains sensitive to OCR quality, visual layout complexity, and position bias in very long documents \cite{borchmann2025arctic}. These architectures preserve page structure and make document-level interaction explicit, but reasoning over many pages must still pass through compressed page summaries, memory states, or sparse attention patterns.

\subsection{Backbone-Centric MLLM Adaptation}
This family treats MP-VRDU as a backbone adaptation problem in which the core MLLM is modified or retrained to encode multi-page inputs, compress page-level multimodal representations, and perform document-level reasoning within a unified inference architecture. Some models rely mainly on instruction tuning, projectors, or adapters to align page images, OCR text, and layout cues with a pretrained language backbone \cite{jia2024leopard,tanaka2024instructdoc}. Other approaches make efficient document representation central to the architecture, using visual token compression, layout-aware encoding, streaming inference, or long-context training to reduce the cost of processing many pages \cite{hu2025mplug,hannan2025docslm,yu2024texthawk2}. Recent work further adapts the backbone toward document-level reasoning through large-scale multi-page training data or evidence-guided reinforcement learning \cite{duan2025docopilot,xiong2026docr1}. These methods reuse the instruction-following and multimodal reasoning abilities of foundation models while introducing targeted document-specific changes. However, document-scale evidence must still be retained inside the backbone, making performance sensitive to context length, compression quality, OCR or visual-text alignment, and preservation of fine-grained evidence across pages.

\subsection{Retriever-Generator}
This family decomposes MP-VRDU into evidence selection and answer generation, where a retrieval module first identifies a compact subset of relevant document content and a generator answers from the selected evidence. Retrieved units may include OCR chunks, pages, visual patches, multimodal elements, or graph nodes, depending on whether the document is represented as text, images, interleaved content, or a structured evidence graph. Retrieval can be performed through coarse-to-fine selection \cite{zhang2024cream}, adaptive visual retrieval \cite{li2025avir}, multimodal dense retrieval \cite{cho2024m3docrag, tanaka2025vdocrag}, end-to-end sparse sampling over paragraphs and figures \cite{xie2024wukong}, graph-based prompting \cite{wang2024knowledge}, or logic-aware retrieval \cite{wu2025molorag}. This architecture improves scalability by limiting the evidence passed to the generator, while the selected evidence provides partial interpretability. However, missed evidence is usually unrecoverable at generation time, and additional retrieved context can introduce noise or attention dilution.

\subsection{Agentic Pipeline}
This family treats MP-VRDU as iterative evidence seeking rather than a fixed retrieve-then-generate pass. In this survey, the category refers to systems that use multi-agent decomposition, tool use, memory, or open-ended control to navigate long documents, inspect evidence, refine queries, and revise answers. Some pipelines distribute retrieval, visual inspection, modal fusion, judgment, and answer synthesis across specialized agents \cite{han2025mdocagent, duan2025m2rag, yu2025visual}. Others use tool-augmented or dynamic iterative workflows to search outlines, fetch sections or pages, crop visual elements, maintain intermediate states, and verify answers \cite{sun2025docagent, zhu2025doclens, zheng2026doc, wang2025vidorag}. These methods improve robustness to scattered, visually fine-grained, or initially missed evidence by gathering additional context before answer generation. However, they introduce higher latency, repeated model calls, dependence on external parsing or tool quality, and greater difficulty in reproducing and evaluating the full decision process.

\section{Multimodal Information Modelling}
MP-VRDU models must model text, visual elements, and layout over multiple pages. Hierarchical models usually encode modalities locally within each page before document-level aggregation. Backbone-centric MLLMs project or compress multimodal page evidence into the backbone context. Retrieval-generator and agentic pipeline models convert modalities into searchable or navigable evidence units.

\subsection{Text Modality}
Text remains the most compact modality for retrieval and reasoning, but in MP-VRDU it is rarely a complete representation of the document. Text may be extracted by OCR or PDF parsers, learned explicitly from pixels, or generated as summaries and queries by an LVLM. The central question is therefore how textual evidence is grounded to pages, layout, and visual regions.

\paragraph{OCR Text as Page-Level Representation.} 
Early hierarchical document transformer models generally treat OCR text as a structured page representation rather than as plain text. Work tokens are paired with bounding boxes, page identifiers, visual patches, and query-conditioned summary or memory tokens. This is so that each page can be encoded locally before document-level aggregation \cite{tito2023hierarchical, dong2024multi}. Later variants add document-global tokens, sparse attention, or layer-wise multimodal fusion to let OCR-based page representations exchange information across pages without fully concatenating all tokens \cite{blau2024gram, borchmann2025arctic}. In this family, OCR text provides the semantic backbone of the page, while layout and visual features preserve grounding.

\paragraph{Text as a Visually Latent Skill.}
Many MP-VRDU models, especially backbone-centric MLLMs, often treat reading as an internal LVLM/MLLM capability rather than a separate OCR preprocessing step. This is typically learned by using text recognition, transcription, or grounding as pretraining and instruction-tuning objectives, often with OCR-derived supervision to avoid expensive human annotation. Some models preserve readable text through high-resolution document pretraining and page-level compression \cite{hu2025mplug}, while others strengthen visual reading by aligning text spans with regions and layout cues \cite{yu2024texthawk2}. This skill can also be invoked selectively after retrieving relevant pages, crops, or visual elements \cite{zheng2026doc, jain2025simpledoc}. Visual retrievers may likewise use OCR-derived supervision during training while remaining image-based at inference \cite{tanaka2025vdocrag}. Thus, text becomes a latent skill used where needed, but remains sensitive to image resolution, compression quality, and grounding ability.

\paragraph{Text for Retrieval and Navigation.}
Retriever-generator and agentic pipeline models often use extracted text as an index for locating evidence rather than as the full model input. This can involve coarse-to-fine OCR chunk retrieval \cite{zhang2024cream}, page-aware text-image chunks with re-ranking \cite{lopez2025enhancing}, hierarchy-preserving section trees \cite{shin2025multidocfusion}, parent-page or cross-page summary indexes \cite{gong2025mhier}, and generated query nodes over multimodal chunks \cite{zhang2026mldocrag}. In agentic workflows, text also acts as a navigation layer: text and image retrieval can propose complementary evidence \cite{han2025mdocagent}, sparse text retrieval can produce evidence chains for comparison with visual extraction \cite{duan2025m2rag}, and outlines or section search can guide later inspection of pages, tables, and figures \cite{sun2025docagent}. Thus, text is best viewed as a searchable scaffold for multimodal evidence gathering, not a stand-alone document representation.

\subsection{Visual Modality}
Visual modelling in MP-VRDU involves deciding when page images should be used as context, evidence or readable input. Unlike single-page VRDU, the multi-page setting makes full visual ingestion impractical. As a result, most models either downsample, compress, retrieve or selectively inspect visual evidence.

\paragraph{Low-Resolution Visual Context and OCR-Grounded Reading.}
Hierarchical models often use low-resolution visual patches as page context while OCR text and layout boxes carry fine-grained reading information \cite{tito2023hierarchical, blau2024gram, borchmann2025arctic}. This keeps page images available for grounding without requiring every page to be encoded at high resolution. Retrieval and agentic systems use a related strategy: OCR or text retrieval narrows the search space, then selected pages, crops, tables, or figures are inspected visually when needed \cite{zhang2024cream, lopez2025enhancing, sun2025docagent, zhu2025doclens, duan2025m2rag}. Low-resolution visual modelling is therefore mainly a scalable grounding strategy.

\paragraph{High-Resolution Visual Reading.}
Backbone-centric MLLMs require a different visual strategy as they must fit visual evidence into the backbone context. High-resolution reading is therefore rarely applied to the full document at once. Instead, these models preserve fine detail through page-level visual compression \cite{hu2025mplug}, dynamic tiling or adaptive token allocation \cite{jia2024leopard, duan2025docopilot}, or fixed-size multimodal page representations that combine visual crops, OCR, and layout features \cite{hannan2025docslm}. High-resolution reading in MP-VRDU is best understood as selective preservation of fine detail rather than full-document high-resolution ingestion.

\paragraph{Visual Embeddings for Retrieval and Navigation.}
Retriever-generator models often represent pages as visual evidence units. Multimodal page embeddings keep text, layout, figures, and tables together during retrieval \cite{cho2024m3docrag}, while dense visual retrievers align page images with document queries before generation \cite{tanaka2025vdocrag}. Other systems score question-page relevance with lightweight visual modules before passing selected pages to an LVLM \cite{li2025avir, kang2024multi}. In agentic pipelines, visual embeddings become part of navigation: thumbnail overviews provide a low-cost document map before high-resolution page fetching \cite{zheng2026doc}, page embeddings can be combined with VLM-generated summaries for iterative retrieval \cite{jain2025simpledoc}, and hybrid agents can alternate between text and visual evidence as reasoning progresses \cite{wang2025vidorag}. Thus, visual representations support both answer generation and evidence acquisition by guiding page selection, region inspection, and iterative document navigation.

\subsection{Layout Modality}
General VRDU methods encode local layout with OCR boxes, 2D positional embeddings, relative spatial biases, layout tokens, or layout-aware prompts. These are covered in prior MLLM-based VRDU surveys \cite{ding2025survey}. In MP-VRDU, the more distinctive problem is modelling document-level structure: page order, section hierarchy, cross-page continuations, references between figures or tables and text, and the broader context of local evidence.

\paragraph{Layout as a Training Objective}
Hierarchical and backbone-centric systems may learn layout sensitivity through objectives that force models to preserve spatial information during compression or cross-page aggregation. Layout-aware denoising trains page representations to reconstruct masked text using both semantic and spatial cues \cite{tito2023hierarchical}. Interleaved text-layout prediction teaches an LLM to treat OCR text and bounding boxes as a joint sequence rather than independent inputs \cite{zhu2025simple}. For OCR-free models, layout-guided visual compression and grounding objectives align high-resolution regions with text spans before reducing page images into compact representations \cite{hu2025mplug, yu2024texthawk2}. Such objectives are especially important in MP-VRDU because cross-page reasoning depends on preserving local spatial evidence after pages have been compressed, retrieved, or summarised.

\paragraph{Layout for Retrieval and Navigation.}
Retriever-generator and agentic systems often model layout as navigable document structure rather than raw geometry. Detected headings and regions can be reconstructed into hierarchy-aware chunks \cite{shin2025multidocfusion}. Parsed pages can be organised into parent-child links and cross-page summaries \cite{gong2025mhier}. Heterogeneous elements such as paragraphs, figures, tables, and equations can become query-centric graph nodes \cite{zhang2026mldocrag}. Graph-based methods further connect passages, pages, and tables through semantic or logical relations, allowing retrieval to follow document structure when surface similarity is insufficient \cite{wang2024knowledge, wu2025molorag}. In these models, layout helps preserve evidence boundaries and guides retrieval and navigation toward where the answer is likely to be located.

\subsection{Multimodal Fusion Overview}
General MLLM-based VRDU surveys already cover common fusion mechanisms such as neural fusion, target-oriented fusion, and prompt-based fusion \cite{ding2025survey}. In this survey, we focus instead on how fusion is adapted to the multi-page setting: when modalities are fused, and how selectively that fusion is applied. This distinction is important as full text-vision-layout fusion across all pages is usually computationally infeasible. MP-VRDU systems therefore either fuse modalities locally within pages, delay fusion until after retrieval or inspection, or selectively fuse only the evidence most relevant to the query.

%     \item Modalities are embedded together and interact via self-attention or cross-attention
%     \item Good for precise alignment between modalities, but scales poorly as page count grows
%     \item Representative models: Hi-VT5, RM-T5, InstructDr, GRAM
% \end{itemize}

\paragraph{Early Fusion.}
Early fusion combines text, visual, and layout features before or during document encoding. In hierarchical models, this is usually local: OCR tokens, bounding boxes, visual patches, and question tokens are fused within each page, then aggregated through page summaries \cite{tito2023hierarchical}, document-level tokens \cite{blau2024gram}, recurrent memory \cite{dong2024multi}, or sparse text-image-layout attention \cite{borchmann2025arctic}. Backbone-centric MLLMs also use early fusion when visual or multimodal features are projected into the LLM token space through adapters, projectors, or document-former modules \cite{tanaka2024instructdoc, duan2025docopilot, jia2024leopard}. Early fusion preserves fine-grained multimodal alignment, but remains costly when many pages must be processed jointly.

% \begin{itemize}
%     \item Modalities are processed more independently first, and only combined after retrieval, ranking, summarisation or page selection. 
%     \item SOme models retrieve OCR text chunks and page images separately, then fuse them only at evidence selection stage or final answer stage
%     \item Scales well for multi-page setting as token overhead is effectively reduced by avoiding fusing whole document jointly.
%     \item Model only fuses small set of candidate pages, crops, chunks or agent outputs
%     \item Problem is modality alignment can be poor, and retrieval/agentic performance is harder when modalities are separate.
%     \item models include MDocAgent, M2RAG, ViDoRAG, CREAM
% \end{itemize}

\paragraph{Late Fusion.}
Late fusion keeps modalities separate during retrieval or inspection and combines them only after candidate evidence has been selected. Text chunks may first identify relevant passages before visual generation \cite{zhang2024cream}, page-image retrieval may keep multimodal evidence fused at the page level \cite{cho2024m3docrag}, and hierarchical indexes may combine parsed text, visual descriptions, parent pages, and cross-page summaries only for retrieved evidence \cite{gong2025mhier}. Agentic systems extend this pattern by letting text, image, table, and page-level evidence be inspected separately before final synthesis or adjudication \cite{han2025mdocagent, duan2025m2rag, zhu2025doclens}. Late fusion scales better to long documents, but depends strongly on retrieval and routing quality.

\paragraph{Staged Fusion.}
Staged fusion combines local alignment with selective document-level reasoning. A model may first fuse modalities within pages or regions, then compress, retrieve, or store selected evidence before performing final cross-page fusion. Some hierarchical and recurrent models approximate this pattern by producing page-level summaries or memories before document-level aggregation \cite{tito2023hierarchical, dong2024multi}, while compact long-document models show how local multimodal evidence can be reduced into fixed-size page representations \cite{hannan2025docslm}. Fully developed staged fusion remains under-explored, but it is a promising direction for MP-VRDU by preserving fine-grained text-vision-layout grounding locally while avoiding full-document multimodal concatenation.

\section{Training Strategies}
Training strategies in MP-VRDU are closely tied to how each architecture controls the cost and complexity of multi-page document understanding. While the previous sections describe how models represent documents and combine text, visual, and layout evidence, this section focuses on how these capabilities are learned.

% 2. Backbone-Centric models - generally following curriculum / multi-stage training

\paragraph{Parameter efficienct tuning on frozen backbones}
A common strategy is to reuse strong pretrained LLMs or LVLMs and update only a small number of task-specific components. Rather than training vision encoders or language backbones end-to-end, these methods freeze most pretrained parameters and introduce lightweight modules such as LoRA, projectors, adapters, layout tokenizers, or task-specific heads. 
LayTokenLLM~\cite{Zhu2025-jb-laytokenllm} exemplifies this by keeping the LLM backbone frozen and introducing a lightweight layout tokenization module together with LoRA-based adaptation. Rather than modifying the architecture, it injects layout information into the input sequence using a compressed representation and a specialized positional encoding scheme that avoids additional context overhead, enabling efficient scaling to long multi-page documents. As an alternative, some work adopts retrieval-augmented strategies, which avoid directly modeling entire documents by selecting only the most relevant content for downstream reasoning. CREAM~\cite{Zhang2024-zs-cream} introduces a coarse-to-fine retrieval framework. It first retrieves candidate text chunks based on embedding similarity, and then applies multi-round LLM-based re-ranking to identify the most relevant evidence. Extending this idea, MoLoRAG~\cite{Wu2025-mi-molorag} incorporates logical reasoning into retrieval by constructing a graph over document pages and performing multi-hop traversal, enabling the model to capture both semantic and logical relevance across pages.

\paragraph{Curriculum / Multi-Stage Training}
Directly adapting single-page models to multi-page objectives is often ineffective and does not scale well, as it fails to capture cross-page dependencies and long-context interactions.
%Either directly adapt single page model, then train on multi-page objectives but doesn't scale well.
Instead, many approaches adopt a progressive training strategy, where models are first trained on simpler local tasks and then gradually extended to more complex multi-page objectives, improving both performance and training efficiency.
%Alternatively train with easier local tasks then move towards harder multi-page objectives (end-to-end training) for better performance and efficiency.
Representative models include DocSLM: image-OCR alignment -> image compression -> document compression -> instructuoin tuning, streaming abstention.
mPLUG-DocOWL2 also.

\paragraph{Self-Supervised Retrieval Training}
Main approach with multi-page is RAG / retrieval, many models train retrieval transformer using techniques such as contrastive learning e.g. VDocRAG uses self-supervised representation compression objectives. 

\paragraph{Reinforcement Learning For Reasoning}
Other main approach with multi-page is agentic iterative reasoning, many models optimize reasoning process e.g. DocR1 uses evidence guided GRPO, DocV* uses SFT + GRPO

Construct knowledge base (structured) from document, then run RAG or agentic iterative retrieval to improve performance.
